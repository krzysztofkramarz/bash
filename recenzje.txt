There are a lot of books on the market, which under the promising title conceal the fiscal documentation of the project. It is as convenient for the reader as it is confusing, because it does not give him grounds to understand what is happening in the kitchen. The book "Hibernate in action", also written by Gavin King, creator of Hibernate, is a very good and in-depth reading. Anyone who thinks that this is just a technical description, a hint on how to implement the framework and use the Hibernate API is wrong. Its advantage is firstly to show the context of applying persistence. It guides the reader through a number of topics, apparently only related to the framework, which describe the problems associated with databases. He explains what differences and problems are between relational databases and the domain model - discusses differences in paradigms. Explains the durability of data, transactions and issues with multithreading, data purity, isolation levels, types of relationships. Turning to the framework as such, the authors write a lot about problems at the interface interface - Java objects. They describe the durability of the object, its life cycles. In addition, we find a lot of information simply, how to use the framework, how to implements individual solutions. The disadvantage of the book is that it was created in 2005, so less than a year before the release of JPA. Therefore, the book shows only the Hibernate API and in version 2.0 - there is no EntityManager, no annotations. Despite these drawbacks, the book is worth reaching for it.
